## 商品从点击购买到购买完成的完整流程

### 1. 商品浏览与选择
- 用户访问首页，系统调用`HomeApiController`的`index`方法
- 通过`HomeServiceImpl`远程调用获取首页数据，包括:
  - 用户提货地址信息
  - 商品分类信息
  - 新人专享商品
  - 热门商品列表
- 用户可以浏览商品，查看商品详情

### 2. 添加商品到购物车
- 当用户点击"加入购物车"，调用`CartApiController`的`addToCart`方法
- `CartInfoServiceImpl.addToCart`实现添加购物车功能:
  - 获取用户ID和购物车Redis键
  - 判断是否首次添加该商品:
    - 如果已存在，更新数量
    - 如果不存在，通过`ProductFeignClient`获取商品信息，新建购物车项
  - 购物车数据存储在Redis中，设置过期时间

### 3. 购物车管理
- 用户可以对购物车进行多种操作:
  - 查看购物车列表(`cartList`方法)
  - 选择/取消选择商品项(`checkCart`方法)
  - 全选/取消全选(`checkAllCart`方法)
  - 批量选择/取消选择(`batchCheckCart`方法)
  - 删除购物车商品项(`deleteCart`方法)
  - 清空购物车(`deleteAllCart`方法)

### 4. 提交订单前确认
- 用户点击"提交订单"，调用`OrderInfoServiceImpl.confirmOrder`方法:
  - 获取用户ID和团长配送点信息
  - 获取已选中的购物车商品列表
  - 生成唯一订单号并存入Redis(防重复提交)
  - 计算商品优惠活动和优惠券信息
  - 返回订单确认信息给用户确认

### 5. 正式提交订单
- 用户确认订单信息后，调用`OrderInfoServiceImpl.submitOrder`方法:
  - 检查订单号是否重复提交(Redis + Lua脚本实现)
  - 获取选中的购物车商品
  - 验证和锁定库存:
    - 将购物车商品转换为库存锁定对象
    - 调用`ProductFeignClient.checkAndLock`锁定库存
    - 锁定失败则抛出异常
  - 保存订单信息(调用`saveOrder`方法):
    - 计算订单金额、优惠金额等
    - 向`order_info`表插入订单基本信息
    - 向`order_item`表插入订单项信息
    - 更新优惠券使用状态
    - 记录用户购物数据到Redis
  - 发送MQ消息删除购物车中已购买商品

### 6. 支付订单
- 订单创建后调用`orderPay`方法进行支付:
  - 验证订单状态
  - 调用`updateOrderStatus`更新订单状态为待发货
  - 通过MQ发送消息执行实际扣减库存操作

### 7. 库存扣减
- MQ消费者接收到库存扣减消息
- 调用`SkuInfoServiceImpl.minusStock`实际扣减库存:
  - 从Redis获取之前锁定的库存信息
  - 遍历每个商品SKU，进行库存扣减
  - 扣减成功后删除Redis中的锁定记录

### 8. 订单完成
- 订单状态流转:
  - 待付款 -> 待发货(支付成功) -> 待收货(发货后) -> 已完成(确认收货)
- 用户可通过`OrderInfoController`查询订单详情和订单列表

### 主要服务间调用关系
1. 首页服务(`service-home`) -> 商品服务(`service-product`)：获取商品信息
2. 购物车服务(`service-cart`) -> 商品服务：获取SKU详情
3. 订单服务(`service-order`) -> 购物车服务：获取已选中购物车项
4. 订单服务 -> 商品服务：锁定和扣减库存
5. 订单服务 -> 活动服务：计算优惠活动和优惠券
6. 各服务间通过Feign进行远程调用，通过RabbitMQ进行异步消息通信

系统采用了分布式架构设计，使用Redis缓存提高性能，使用分布式锁保证库存操作的原子性，通过消息队列实现服务间的异步通信，保证了系统的高可用性和数据一致性。





## 订单提交后支付业务流程

### 1. 用户发起支付请求

当用户正式提交订单后（通过`OrderInfoController`中的`submitOrder`方法），系统会返回订单ID（orderId）。用户可以在订单详情页面看到订单并选择支付。

### 2. 调用微信支付接口

1. **创建支付预订单**：
   - 用户点击支付按钮时，前端调用`WeixinController`中的`createJsapi`方法
   - 该方法通过订单号（orderNo）查询或创建支付记录（`PaymentInfo`对象）
   - 调用`WeixinServiceImpl.createJsapi`方法，该方法:
     - 通过`PaymentInfoService.savePaymentInfo`保存支付信息记录
     - 封装微信支付所需参数（appid、商户号、订单号、金额等）
     - 使用`HttpClient`调用微信支付系统生成预付单
     - 返回支付所需的参数（timeStamp、nonceStr、package、signType、paySign等）

2. **前端展示支付界面**：
   - 前端使用返回的支付参数，调用微信JSAPI发起支付请求
   - 用户在微信支付界面完成支付操作

### 3. 查询支付状态

1. **轮询查询支付结果**：
   - 前端通过调用`WeixinController`的`queryPayStatus`方法，轮询查询支付结果
   - 该方法会调用`WeixinServiceImpl.queryPayStatus`，向微信支付系统查询订单支付状态
   - 返回三种可能状态：
     - 支付失败：返回错误信息
     - 支付成功：处理支付成功逻辑
     - 支付中：继续等待

2. **支付成功处理**：
   - 当查询到支付状态为"SUCCESS"时，调用`PaymentInfoService.paySuccess`方法
   - 该方法会更新支付记录的状态为"已支付"
   - 通过RabbitMQ发送支付成功消息（使用`MqConst.EXCHANGE_PAY_DIRECT`交换机和`MqConst.ROUTING_PAY_SUCCESS`路由键）

### 4. 消息队列处理支付成功

1. **消息监听与处理**：
   - `OrderReceiver`类使用`@RabbitListener`注解监听支付成功消息
   - 监听配置绑定到`MqConst.QUEUE_ORDER_PAY`队列、`MqConst.EXCHANGE_PAY_DIRECT`交换机和`MqConst.ROUTING_PAY_SUCCESS`路由键
   - 收到消息后，调用`OrderInfoService.orderPay`方法，传入订单号（orderNo）

2. **订单状态更新**：
   - `OrderInfoService.orderPay`方法完成以下操作：
     - 查询订单信息，检查是否已支付
     - 调用`updateOrderStatus`方法更新订单状态为"待发货"
     - 发送消息到RabbitMQ扣减库存（使用`MqConst.EXCHANGE_ORDER_DIRECT`交换机和`MqConst.ROUTING_MINUS_STOCK`路由键）

### 5. 库存扣减

在支付成功后，系统通过消息队列异步处理库存扣减：
- 商品服务会监听库存扣减消息
- 执行实际的库存扣减操作（`SkuInfoServiceImpl.minusStock`方法）
- 从Redis获取之前锁定的库存信息并进行实际扣减

### 6. 支付流程完成

至此，订单支付流程完成，订单状态变为"待发货"，商品库存被实际扣减，系统可以进入后续的发货、收货流程。

### 关键点总结

1. **支付触发**：由用户在前端页面点击支付按钮触发
2. **微信支付**：系统通过HTTP请求调用微信支付接口生成支付订单
3. **状态查询**：前端通过轮询方式查询支付状态
4. **异步处理**：使用RabbitMQ消息队列实现支付成功后的异步处理
5. **状态流转**：订单状态从"未支付"变为"待发货"
6. **库存操作**：支付前锁定库存，支付成功后才实际扣减库存

整个支付流程采用了分布式架构设计，通过消息队列实现了系统间的解耦，保证了数据的一致性和系统的可靠性。
